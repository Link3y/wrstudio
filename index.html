Warak Studio

https://warak.studio

키보드, 마우스, 터치 조작을 모두 이용하여 캐릭터 이동과 맵 이동이 가능한
RPG게임 컨셉의 음악가 포트폴리오 사이트.
프론트에서 접근과 조작 가능한 맵과 이벤트 등의 모든 요소는 관리자 모드의 빌더를 이용해 구성 가능하도록 했다.

PHP / Codeigniter 4
프레임웍 선정에 있어 최초에 라라벨을 고려 하였다가,
서비스 규모과 기능으로 보아 코드이그나이터가 개발과 운영에 있어 더욱 효율적일 것으로 판단함.
AWS DynamoDB
웹서버와 DB 모두 AWS 사용.
SQL을 사용하지 않고 DynamoDB를 사용하며, 코드이그나이터의 캐싱 라이브러리를 활용해 DB에 직접 쿼리를 최소화 시킴.
HTML5 Canvas
RPG게임과 같이 캐릭터를 조작해야 하는 기획 특성 상 Canvas에 웹게임과 유사한 형태로 구현.
별도의 게임 엔진을 사용하지 않고 순수 JS로 구현했다.
JS ES6
유지보수가 빈번하지 않을 서비스 특성을 고려해, 작업 숙련도가 가장 높고 퍼포먼스가 높은 Vanilla JS 이용

Warak Studio 관리자모드 소개

관리자 로그인

IP whitelist 등록과 비밀번호 입력의 2중 보안을 적용했다.
특정 경로의 페이지에 접근 후, 토큰 파일을 업로드해 접속 IP를 whitelist에 등록해야 관리자 페이지의 로그인 폼에 접근이 가능함.

서비스 설정

패스워드, 화이트리스트, SMTP, favicon 등등 여러가지를 설정 가능하게 하여, 이용자가 개발자를 통하지 않고 최대한 스스로 운영 가능하도록 했다.

전역 설정

캔버스의 크기, 캐릭터 디자인과 이동속도 등 전역설정.
클라이언트 요구에 따라 종/횡 이동속도를 다르게 지정 가능하도록 함.

맵 빌더

각 맵 생성/수정, 오브젝트 배치, 충돌영역 지정, 사운드 재생, 애니메이션 재생, 페이지 팝업 등의 이벤트를 모두 설정 가능하게 함.

소스 관리

각 맵에 사용될 배경 이미지, 배치할 수 있는 오브젝트, 효과음 파일 등등을 이용자가 직접 관리 가능하도록 함.

게시판 설정

각 게시판의 설정과 사용 필드도, 요구조건 안에서 이용자가 최대한 유연히 운영 가능하도록 했다.
DynamoDB를 사용하니 이런 식으로 활성 컬럼을 임의로 설정하는 방식이 효율적으로 적용 가능했다.
